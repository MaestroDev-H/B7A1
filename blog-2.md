# DRY কোড: TypeScript-এর `Pick` এবং `Omit` ইউটিলিটি টাইপের ব্যবহার

সফটওয়্যার ডেভেলপমেন্টের একটি অন্যতম জনপ্রিয় মূলনীতি হলো **DRY (Don't Repeat Yourself)**। অর্থাৎ, একই কোড বা টাইপ বারবার না লিখে কীভাবে রিইউজ করা যায়। টাইপস্ক্রিপ্টে ইন্টারফেস বা টাইপ ডুপ্লিকেশন এড়াতে `Pick` এবং `Omit` ইউটিলিটি টাইপ দুটি অসাধারণ ভূমিকা পালন করে।

---

## ১. টাইপ ডুপ্লিকেশনের সমস্যা

ধরে নেওয়া যাক, আমাদের একটি বড় `User` ইন্টারফেস আছে:

```typescript
interface User {
  id: string;
  name: string;
  email: string;
  role: string;
  createdAt: Date;
}
```

এখন আমাদের এমন একটি ফাংশন দরকার যেখানে শুধুমাত্র ইউজার প্রোফাইলের জন্য নাম এবং ইমেইল প্রয়োজন। আমরা যদি আবার নতুন করে একটি ইন্টারফেস তৈরি করি, তবে তা কোড ডুপ্লিকেশন তৈরি করবে। ভবিষ্যতে মূল `User` এ পরিবর্তন আসলে সাব-টাইপগুলোতেও ম্যানুয়ালি চেঞ্জ করতে হবে, যা অত্যন্ত বিরক্তিকর ও ভুলের সুযোগ বাড়ায়।

## ২. সমাধান ১: `Pick` ব্যবহার করে টাইপ স্লাইসিং

`Pick` ইউটিলিটি টাইপ আমাদের একটি বড় ইন্টারফেস থেকে নির্দিষ্ট কিছু প্রপার্টি বেছে নিয়ে নতুন একটি টাইপ তৈরি করতে সাহায্য করে।

**উদাহরণ:**

```typescript
type UserProfile = Pick<User, "name" | "email">;

const profile: UserProfile = {
  name: "Habibur Rahman Rahel",
  email: "rahel.ahmedr@gmail.com"
};
```

## ৩. সমাধান ২: Omit ব্যবহার করে অপ্রয়োজনীয় অংশ বাদ দেওয়া

অনেক সময় আমরা বড় ইন্টারফেসের প্রায় সব প্রপার্টিই ব্যবহার করতে চাই, শুধু দু-একটি বাদ দিতে চাই। এই ক্ষেত্রে `Omit` সেরা সমাধান। এটি নির্দিষ্ট কিছু প্রপার্টিকে বাদ দিয়ে বাকিগুলো নিয়ে নতুন টাইপ তৈরি করে।

**উদাহরণ:**

আমরা যদি রেজিস্ট্রেশনের সময় `id` এবং `createdAt` বাদ দিয়ে বাকি সব ডেটা নিতে চাই:

```typescript
type UserRegistrationData = Omit<User, "id" | "createdAt">;

const newUser: UserRegistrationData = {
  name: "Habibur Rahman Rahel",
  email: "rahel.ahmedr@gmail.com",
  role: "admin"
};
```

## কীভাবে এটি আমাদের কোডকে DRY রাখছে?

- একক উৎস **(Single Source of Truth)**: মূল `User` ইন্টারফেসটিই সমস্ত টাইপের অভিভাবক।

- সহজ রক্ষণাবেক্ষণ: ভবিষ্যতে যদি `name` -এর টাইপ পরিবর্তন করে অবজেক্ট করা হয়, তবে স্বয়ংক্রিয়ভাবে `Pick` ও `Omit` দিয়ে তৈরি সব টাইপ আপডেট হয়ে যাবে। কোনো ম্যানুয়াল ডুপ্লিকেট চেঞ্জের প্রয়োজন পড়বে না।

## উপসংহার

ছোট ছোট প্রপার্টি স্লাইস করার জন্য আলাদা করে শত শত ইন্টারফেস না লিখে, মূল ইন্টারফেস থেকে `Pick` এবং `Omit` ব্যবহার করা কোডকে অনেক বেশি গোছানো, রিডেবল এবং মেইনটেইনেবল করে তোলে।
